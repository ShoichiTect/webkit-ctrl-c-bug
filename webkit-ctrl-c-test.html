iPadOS Ctrl+C keyCode=13 — Layer Diagnosis

Objective: Identify whether the bug is in UIKit (system shortcut interception) or WebKit (DOM event construction).

Verification checklist

  1. Does copy event fire after Ctrl+C? → If yes, UIKit routed it as Copy
  2. Does preventDefault() on keydown stop the copy event? → If yes, DOM prevention propagates up
  3. Does preventDefault() also fix keyCode? → If yes, keyCode corruption is downstream of Copy routing
